I found out that you can use the function "eval" to evaluate a string of integers and operators natively. The result is calculated using the correct order of operations. As opposed to Googles calculator app that reads everything from left to right. Therefore an expression like 1+2*5 will come out correctly as 11 on my calculator and not 15.
